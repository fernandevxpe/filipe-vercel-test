Smoke test estático — validar Vercel + Git sem Next.js
======================================================

SOBRE "npx plugins add vercel/vercel-plugin"
------------------------------------------
Isso é plugin para Cursor / agentes de IA (documentação Vercel no editor).
NÃO resolve NOT_FOUND nem altera o deploy. Opcional; podes ignorar.

1) Na Vercel: "Add New Project" → importa o MESMO repositório GitHub (filipe-estudo).

2) ANTES de fazer deploy, abre "Configure Project":
   - Root Directory: deploy-smoke  (OBRIGATÓRIO — pasta deploy-smoke, não a raiz do repo)
   - Framework Preset: Other
   - Build Command: npm run build   (ou deixa vazio se a Vercel ler do vercel.json)
   - Output Directory: VAZIO  (a saída é a própria raiz, com index.html)
   - Install Command: npm install   (ou vazio; há package.json sem dependências)

3) Depois do deploy, abre o deployment → separador "Building" / Build Logs:
   - Deve aparecer o caminho com .../deploy-smoke ou "Running build in" dentro dessa pasta.
   - Se só aparecer a raiz do monorepo SEM deploy-smoke, o Root Directory está mal.

4) Faz Deploy e abre o URL que a Vercel der. Deves ver a página escura com "Deploy smoke OK".

Se ISTO falhar (NOT_FOUND ou 404), o problema é conta/equipa/domínio na Vercel, não o código Next em web/.

Se ISTO funcionar mas web/ não, o problema está no build Next (Root web/, dependências, etc.) — aí migramos ou corrigimos só o web/.
